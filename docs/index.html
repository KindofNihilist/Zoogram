About the app

This app offers a seamless platform for sharing photos of animals, enabling users to like, bookmark, comment on posts, and follow other users. When creating a new post, users can crop images, apply various filters and editing effects, or simply use the auto-enhance feature for quick adjustments. The app features an Activity system that notifies users of recent interactions with their posts and profile. Built with MVVM architecture in mind, it leverages protocols extensively to ensure testability and maintainability.

Frameworks used

  • Swift
  • UIKit
  • MetalKit
  • Photos
  • FirebaseAuth
  • FirebaseStorage
  • FirebaseDatabase
  • SDWebImage

Key Features

Crop View

Allows user to crop an image with the use of a responsive, self-aligning view that enables simultaneous panning and pinching. Implemented by using pan and pinch UIGestureRecognizers, with calculations based on the current scale and the deltas of the frames.

Animated Post Insertion

When creating a new post, the user will see a progress bar displaying the picture being uploaded, which will seamlessly transform into the post UITableViewCell once the upload is complete.

Input View with Character Limit

In order to stop users misusing comment section the custom input accessory view was designed to feature a character limit with visual feedback, that allows user to see if the limit is being aproached or has been exceeded.

Animated "Nose" Button

Button resembles a cat's sniffing nose and is made by creating multiple shapes and gradients for different states with the use of UIBezierPath and control points. To enhance the experience, a custom designedhaptic pattern plays on each tap, reminding user of a purring cat.

Photo Edditor

Before making a post, the user can edit a picture in the Photo Editor, which features an auto-enhance option, a set of commonly used parameters, and custom pre-made filters. The user can mix and match all three options and blend the applied filter to the desired value. This functionality is implemented using CIFilters for image manipulations and MetalKit for a fast, responsive preview of the results. Since the order in which parameters and filters are applied can drastically affect the final outcome, a queue was created to hold the order in which filters were applied.

Activity System

Observes activity events at the endpoint and delivers them to the Activity tab, which displays them neatly in a table view with separate cells for each kind of activity event. Initially, every event has a ‘seen’ status set to false, highlighting unseen events with a blue tint. If the event kind is ‘post liked’ or ‘commented on post’ the cell displays the associated post, allowing quick navigation to it. Additionally, tapping on a ‘commented on post’ kind of event will open the post, scroll to the comment, and highlight it.

Offline capabilites

When making a comment, the user can see an indicator showing if it has been posted. If there is no connection and the user attempts to leave the comment section while there is a pending comment, they will be alerted that the comment will be posted once the connection is reestablished. The alert provides two options: one to close the alert and exit the comment section, and the other to ensure that this alert won't be displayed again in the future. If the user deletes a pending comment, it will not be posted.

Interactive Navigation Bar

To fit more content on the screen, an interactive navigation bar was designed. It monitors the scroll direction and hides or shows accordingly with the movement of a finger. If the user interrupts the scroll before the navigation bar fully enters one of its states, it will automatically animate to the state corresponding to the latest scroll direction.

Sticky Slider

If the user lifts their finger above the predetermined ‘dead zone’ the slider will snap to the default value. Since different CIFilters use varying value ranges, the ‘dead zone’ is calculated separately for each parameter and matches 4% of the slider’s total width. To make it more intuitive, reaching a minimum, maximum, or default value triggers a haptic response.

Connection Monitor

The app monitors internet connection and notifies user of the changes. For every action requiring a network request, an error handler is implemented to display a notification, alert, or retry button, depending on the context.

Registration

The registration process is handled in a card-like manner, checking and verifying entered data at each step. Each step has a corresponding error handler with localized error descriptions, so if the username is already taken, the email format is incorrect, the password is too short, etc., the user will be notified. Once registration is complete, the screen will seamlessly transition to the app’s main view.

Discover Tab

This tab allows user to search for other users and browse recent posts from users they’re not following. Like every other UITableView and UICollectionView in this app, it features a pagination system, which is crucial for this type of app.

Zoom Transition

When running on iOS 18.0 or later, the app utilizes the new ‘Zoom’ transition style, seamlessly transitioning from a UICollectionView cell to a UITableView cell. Furthermore, it is designed to track the most visible post on the screen so it can transition back to the corresponding post’s cell in the collection view.
